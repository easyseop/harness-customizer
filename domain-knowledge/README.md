# Domain Knowledge 주입 가이드

> 커스터마이징 봇이 사내 전문 도메인을 모를 때 여기에 파일을 추가한다.
> 봇은 질문을 받으면 이 폴더를 먼저 확인하고, 관련 파일이 있으면 참고하여 답변한다.

---

## 파일 추가 방법

1. 아래 템플릿을 복사하여 `[도메인명].md` 파일 생성
2. 내용 작성 후 저장
3. 봇에게 다시 질문하면 자동으로 참고함

## 파일명 규칙

| 도메인 | 권장 파일명 |
|--------|-----------|
| 내부망 시스템 구성 | `internal-systems.md` |
| 보안 정책/기준 | `security-policy.md` |
| 공통 프레임워크 | `common-framework.md` |
| 인증/접근제어 | `auth-policy.md` |
| 파일 전송 솔루션 | `file-transfer.md` |
| MQ/미들웨어 | `middleware.md` |
| 배포/운영 정책 | `deployment-policy.md` |
| 감사/규정 요건 | `compliance.md` |

## 파일 템플릿

아래를 복사해서 새 파일 만들기:

```markdown
# [도메인명]

## 개요
[이 도메인이 무엇인지 한 줄]

## 핵심 제약사항
- [제약 1]
- [제약 2]

## 사내 표준/규칙
- [표준 1]
- [표준 2]

## 봇이 알아야 할 것
[커스터마이징 봇이 이 도메인 관련 질문에 답할 때 반드시 참고해야 할 내용]

## 관련 시스템/도구
| 시스템 | 용도 | 내부 대체 여부 |
|--------|------|-------------|
| [시스템명] | [용도] | [예: 외부 RabbitMQ → 내부 IBM MQ] |
```
